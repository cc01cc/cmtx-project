## Overview

Fixes common LLM shell command errors: paths with unwanted spaces, unquoted variables, missing quotes around paths, and malformed package names. Includes MiniMax M2.5 parameter tuning.

## When To Use This Skill

Use this skill when:

- **LLM generates broken shell commands** with spaces in paths (e.g., `cd /path / to / dir`)
- **Paths with spaces are unquoted** (e.g., `cd /my project` instead of `cd "/my project"`)
- **pnpm package names have spaces** (e.g., `pnpm -F @cmtx / core build`)
- **Variables aren't protected** (e.g., `find $path` instead of `find "$path"`)
- **You're debugging MiniMax M2.5 command errors** and need parameter tuning

## Required Inputs

1. **Context**: The broken or suspicious shell command generated by LLM
2. **Goal**: What the command should accomplish
3. **Path context**: Does the command work with paths that have spaces?
4. **Model info**: Are you using MiniMax M2.5 or another model?

## Quick Fix: 4 Core Rules

| Rule                 | ❌ Wrong                     | ✅ Correct                  |
| -------------------- | ---------------------------- | --------------------------- |
| Paths without spaces | `cd /path / to / dir`        | `cd /path/to/dir`           |
| Paths with spaces    | `cd /my project`             | `cd "/my project"`          |
| Variable references  | `find $path -name "*.js"`    | `find "$path" -name "*.js"` |
| pnpm package names   | `pnpm -F @cmtx / core build` | `pnpm -F @cmtx/core build`  |

## Fix Workflow

### Step 1: Identify the Error Pattern

```bash
# Unwanted spaces in path?
❌ find /workspace / packages / core -name "*.ts"

# Unquoted path with spaces?
❌ cd /home/user/My Project

# Unprotected variable?
❌ find $project_dir -name "*.js"

# Package name with spaces?
❌ pnpm -F @cmtx / core build
```

### Step 2: Apply the Correct Pattern

| Pattern          | Fix          | Example                     |
| ---------------- | ------------ | --------------------------- |
| **No spaces**    | Remove them  | `cd /path/to/dir`           |
| **Has spaces**   | Add quotes   | `cd "/my project"`          |
| **Variable**     | Use `"$var"` | `find "$path" -name "*.js"` |
| **pnpm package** | No spaces    | `pnpm -F @cmtx/core build`  |

### Step 3: Verify Before Execution

```bash
# Preview what will run
echo "Command: find \"$path\" -name \"*.js\""

# Or use set -x for full trace
set -x
your_command_here
set +x
```

## For MiniMax M2.5 Users

If LLM keeps generating path errors, update model parameters in `kilo.jsonc`:

```json
"MiniMax-M2.5": {
  "options": {
    "temperature": 0.3,           // ↓ from 0.7
    "top_p": 0.8,                 // ↓ from 0.9
    "frequency_penalty": 0.5,     // ← new
    "presence_penalty": 0.3       // ← new
  }
}
```

**Why**: Lower temperature + penalties prevent random space injection and repeated errors.

**Alternative models**: Try `qwen3.6-plus` (more stable) or `GLM-5-Pro` (stronger reasoning).

## Checklist: Is Your Command Correct?

Before running any generated command, verify:

- [ ] No **unwanted spaces** in paths (e.g., `/path / to / dir`)
- [ ] All **paths with spaces** are quoted
- [ ] **All variables** use `"$var"` format (not `$var`)
- [ ] **pnpm package names** have no spaces (e.g., `@cmtx/core`, not `@cmtx / core`)
- [ ] **Quotes match** (no `"...'` or `'..."` mismatches)
- [ ] Command **reads logically** and looks executable
- [ ] If unsure, use `echo` or `set -x` to preview

## Debugging Edge Cases

### Path not found?

```bash
# Step 1: Verify the path exists
ls -la "/path/to/check"

# Step 2: Check for hidden characters
echo "/path/to/check" | od -c
```

### Command still fails?

```bash
# Use set -x to see what's actually running
set -x
your_command
set +x

# Or use bash -x to debug a script
bash -x your_script.sh
```
