# Plan Mode Skill

## Purpose

Ensures Markdown plan documents in `.kilo/plans/` have consistent metadata structure with valid YAML frontmatter. When activated, this skill guides the user through planning phases (exploration, synthesis, documentation) while automatically validating YAML metadata compliance. File naming is delegated to kilocode.

## When To Use

Use this skill when:

- Starting a **new planning session** for features, architecture design, or major refactoring
- You need **structured metadata** (title, date) in all plans in `.kilo/plans/`
- You want **automatic validation** of YAML frontmatter compliance
- You prefer **read-only exploration** before writing decisions
- You want **optional tool-level enforcement** via Kilo rules

Note: File naming is handled by kilocode; this skill focuses on metadata standardization only.

## Required Inputs

Before activation, gather these details:

1. **Plan Title**: What is the subject? (e.g., "Feature Design", "Architecture Review")
2. **Exploration Scope**: Which parts of codebase are relevant?
3. **Output Location**: File will be in `.kilo/plans/` (name auto-generated by kilocode)

## Workflow

### Phase 1: Setup

1. Confirm plan **title** (what is the subject?)
2. Note that file name will be auto-generated by kilocode
3. Plan file will be created in `.kilo/plans/` directory

### Phase 2: Exploration (Read-Only)

1. Use read-only tools only: `read_file`, `grep_search`, `semantic_search`
2. Gather context from codebase
3. Use `runSubagent` (Explore agent) for focused analysis
4. Ask clarifying questions if needed

### Phase 3: Synthesis

1. Review findings from Phase 2
2. Identify critical files to modify
3. Plan verification approach (how to test)

### Phase 4: Document Plan

1. Create file: `.kilo/plans/PLAN-<ID>-<TITLE>.md`
2. Include YAML frontmatter:
    ```yaml
    ---
    title: "Plan Title Here"
    date: 2026-04-15T21:51:16+08:00
    ---
    ```
3. Document:
    - Recommended approach with rationale
    - Critical files to be modified
    - Verification section

### Phase 5: Validation

System automatically validates:

- ✅ YAML frontmatter has `title` and `date` fields
- ✅ `date` is ISO 8601 format with timezone
- ✅ File located in `.kilo/plans/` directory

Note: File naming is handled by kilocode separately.

### Phase 6: Handoff

Plan is ready for implementation in separate sessions.

## File Location

All plan files must be in:

```
.kilo/plans/
```

File naming is auto-generated by kilocode. Examples:

- `.kilo/plans/1776264273233-neon-garden.md`
- `.kilo/plans/1776264273234-arctic-wolf.md`

## Metadata Requirements

Every plan file MUST have YAML frontmatter:

```yaml
---
title: "Plan title here"
date: 2026-04-15T21:51:16+08:00
description: "Optional longer description"
---
```

Required fields:

- `title`: Document title (string)
- `date`: ISO 8601 format with timezone (e.g., `2026-04-15T21:51:16+08:00`)

Optional fields:

- `description`: Longer summary
- `tags`: Related topics
- `status`: draft | active | completed

## Validation

### Automatic Checks

Run `python3 .kilo/skills/plan-mode/scripts/validate-plan.py`:

```bash
python3 .kilo/skills/plan-mode/scripts/validate-plan.py .kilo/plans/PLAN-001-test.md
```

Validates:

- YAML frontmatter presence
- Required fields (title, date)
- Date format (ISO 8601)
- File location

### Manual Checklist

Before marking plan complete:

- [ ] YAML frontmatter includes title and date
- [ ] Date in ISO 8601 format with timezone
- [ ] File in `.kilo/plans/` directory
- [ ] Plan has rationale section
- [ ] Plan has verification section
- [ ] All references resolved (no placeholders)

Note: File naming is auto-managed by kilocode

## Optional: Tool Enforcement

For stricter control, add Kilo rule to `/workspace/.kilo/kilo.jsonc`:

```jsonc
{
    "rules": [
        {
            "name": "plan-mode-strict",
            "applyWhen": {
                "fileActive": ["**/PLAN-*.md"],
                "inDirectory": ".kilo/plans/",
            },
            "toolRestrictions": {
                "prohibited": ["run_in_terminal", "run_task"],
                "phaseRules": {
                    "exploration": {
                        "allowed": [
                            "read_file",
                            "grep_search",
                            "semantic_search",
                        ],
                    },
                    "documentation": {
                        "allowed": ["create_file", "replace_string_in_file"],
                    },
                },
            },
            "precedence": 99,
        },
    ],
}
```

See `.kilo/rules/plan-mode-strict.md` for details.

## Troubleshooting

### YAML frontmatter missing

**Error**: `Missing required frontmatter`

**Solution**:

- Add YAML block at top of file:
    ```yaml
    ---
    title: "Your Title"
    date: 2026-04-15T21:51:16+08:00
    ---
    ```

### Date format invalid

**Error**: `Invalid date format`

**Solution**:

- Use ISO 8601 with timezone: `2026-04-15T21:51:16+08:00`
- Use `date` command: `date -Iseconds`
- Or import from reference script

### File in wrong location

**Error**: `Plan file not in .kilo/plans/`

**Solution**:

- Move file to `.kilo/plans/` directory
- Ensure `.kilo/plans/` directory exists
