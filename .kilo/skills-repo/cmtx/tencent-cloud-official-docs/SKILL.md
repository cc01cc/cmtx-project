在处理腾讯云相关任务时，先基于腾讯云官方文档建立事实基础，再给出建议、步骤、命令、架构判断或风险提示。

## When to Use This Skill

在以下场景使用本 skill：

- 用户提到腾讯云或 Tencent Cloud 产品
- 用户询问 CVM、COS、VPC、CLB、CAM、TencentDB、TDSQL、TKE、SCF（云函数）、CDN、WAF、DNS、日志服务（CLS）、消息队列（CMQ/TDMQ）、云直播、实时音视频等组件
- 用户需要 API、SDK、CLI、Terraform、控制台配置步骤
- 用户需要最新计费、配额、地域、可用区、版本差异、实例规格、权限策略或产品限制
- 用户要求给出架构设计、迁移方案、运维排障、安全基线或成本优化建议
- 用户给出旧资料、截图、博客或记忆性说法，需要验证是否仍然有效

## Required Inputs

开始前尽量确认以下输入：

1. 目标产品或服务名称。
2. 任务类型：概念解释、配置操作、编码开发、故障排查、架构设计、成本或权限问题。
3. 约束条件：地域、网络环境、语言/SDK、操作系统、资源规模、合规要求。
4. 用户是否要求特定时间范围内的最新信息。

若以上信息缺失且会影响结论，先提出最少量的澄清问题；如果不影响先做初步检索，则先查官方文档再补问。

## Core Rule

涉及腾讯云事实性信息时，默认执行以下约束：

1. 必须先查腾讯云官方文档，再输出关键结论。
2. 优先使用产品文档、API 文档、SDK 文档、最佳实践、FAQ、计费与限制页面。
3. 发现第三方博客、论坛、旧教程与官方文档冲突时，以官方文档为准。
4. 若无法获取官方文档或官方文档未覆盖该细节，必须明确说明不确定性，禁止把猜测写成事实。
5. 输出中应标明关键结论来自官方文档，并指出产品名、功能名、限制项或页面主题，便于后续复核。

详细规范见 [official-docs-policy.md](./references/official-docs-policy.md)。

## Workflow

### Step 1. 识别任务类型

先判断当前请求属于哪一类：

- 产品能力与概念解释
- 控制台配置或开通步骤
- API/SDK/CLI/Terraform 开发接入
- 账单、计费、规格、配额与地域限制
- 权限、安全、CAM 策略或网络边界
- 排障、监控、日志与错误码定位
- 架构设计、选型比较、迁移与最佳实践

### Step 2. 锁定官方来源

至少优先检查以下来源：

1. 腾讯云产品文档入口 `https://cloud.tencent.com/document/product`
2. 对应产品的使用文档与最佳实践
3. API 参考文档 `https://cloud.tencent.com/document/api`
4. 对应语言的 SDK 文档或官方示例（腾讯云 SDK 中心：`https://cloud.tencent.com/document/sdk`）
5. 计费、配额、限制、版本说明、发布记录页面
6. 腾讯云开发者社区官方技术文章（仅作补充，不替代产品文档）

如果主题涉及多个产品，分别确认每个产品的官方页面，不要把跨产品默认行为混为一谈。

### Step 3. 提取最新事实

从官方文档中优先提取以下信息：

- 当前产品名称、版本、功能边界
- 前置条件、依赖组件与开通要求
- 最新配置步骤、参数说明、默认值与限制
- 权限要求（CAM 策略）、网络要求（VPC/安全组/ACL）、地域限制与计费影响
- 常见报错、限制条件、配额上限与官方建议

若文档内容存在日期、版本或地域差异，必须在回答里指出。

### Step 4. 形成答案

输出时遵守以下顺序：

1. 先给结论或建议。
2. 再给基于官方文档整理的关键步骤或代码。
3. 最后列出限制、风险、前提条件和可能的版本差异。

如果用户要代码：

- 先确认所用语言、SDK 与部署方式。
- 避免引用过时接口名、旧版 SDK 或废弃参数。
- 生成示例后，提醒用户根据官方文档核对版本与地域支持。
- 腾讯云 Go/Python/Java/Node.js SDK 均有独立版本文档，务必确认是否使用最新版。

如果用户要架构建议：

- 把"事实"与"建议"分开写。
- 事实部分必须基于官方文档。
- 建议部分可以结合通用工程经验，但要明确哪些是经验判断。

### Step 5. 冲突与不确定性处理

当遇到以下情况时，必须显式说明：

- 官方文档之间表述不一致
- 文档未明确当前地域、版本或套餐是否支持
- 用户给出的做法来自旧教程或第三方文章
- 某些能力可能已下线、改名或迁移入口（腾讯云产品改版频繁，需注意）

此时应优先输出：当前已确认信息、待确认项、建议复核的官方页面主题。

## Tips & Recommended Practices

以下技巧可提升腾讯云文档查询质量：

### 文档检索技巧

- **产品文档直达**：`https://cloud.tencent.com/document/product/<产品 ID>`，产品 ID 可从文档导航或 URL 中提取。
- **API Explorer 辅助**：腾讯云提供 API Explorer（`https://console.cloud.tencent.com/api/explorer`），可在线调试接口、生成 SDK 代码，遇到接口调用问题时先用 API Explorer 验证参数。
- **错误码速查**：在 API 参考文档的"公共错误码"和"产品错误码"页单独核查；不要把其他产品线的错误码混用。
- **SDK 版本锁定**：腾讯云 3.0 SDK（`tencentcloud-sdk-*`）为当前主流，1.x/2.x 已不再推荐；检索示例时注意版本前缀。

### 腾讯云特有注意点

- **地域与可用区**：腾讯云部分高级功能仅限特定地域（如金融地域需单独申请），查文档时务必确认地域覆盖。
- **主账号与子账号 (CAM)**：大多数操作需要子账号持有对应 CAM 权限策略，查文档时同步确认所需权限动作（`qcs::`）。
- **标签与项目**：腾讯云有"标签"和"项目"两套资产分类体系，计费分账文档在不同产品下入口不一。
- **备案要求**：通过腾讯云大陆节点对外提供 HTTP/HTTPS 服务前需 ICP 备案，相关限制在产品文档的"使用限制"中说明。
- **免费额度**：云函数（SCF）、COS 等产品有每月/每年免费额度，核对计费文档时注意区分"免费额度"与"按量计费起点"。

### 架构设计建议

- 优先参考腾讯云官方"最佳实践"和"架构中心"（`https://cloud.tencent.com/act/pro/architecture`）页面。
- 使用腾讯云"云上迁移"系列文档作为迁移场景的基础素材。
- 安全合规方面，可查阅腾讯云"安全合规"专区，了解等保、SOC、ISO 等认证覆盖范围。

## Validation Checklist

输出前自检：

- 是否已经优先查阅腾讯云官方文档
- 是否区分了官方事实与个人建议
- 是否标出了限制、版本、地域、计费或权限前提
- 是否避免把过时博客或经验贴当成当前事实
- 若存在不确定性，是否明确说明
- SDK 示例是否基于腾讯云 3.0 SDK 或当前推荐版本

## Troubleshooting

| Problem        | Likely Cause                   | Fix                                                |
| -------------- | ------------------------------ | -------------------------------------------------- |
| 回答基于旧知识 | 未先检索官方文档               | 先回到产品文档入口重新确认                         |
| 配置步骤不适用 | 忽略地域、版本或控制台改版     | 补查当前地域/版本文档与发布说明                    |
| API 示例报错   | SDK 版本、接口版本或权限不匹配 | 查 API 参考、SDK 版本说明与 CAM 权限要求           |
| 成本判断偏差   | 忽略计费项或免费额度变化       | 补查计费文档、定价页与规格限制                     |
| 方案泛泛而谈   | 未拆分产品级事实与架构建议     | 先列官方事实，再给工程建议                         |
| 权限报错       | 未确认 CAM 策略权限动作        | 查对应产品的 CAM 授权指引，核对 Action 与 Resource |

## Example Prompts

- 帮我看看腾讯云 TKE 现在怎么给集群配置 Ingress，先查官方最新文档再回答。
- 用最新腾讯云官方资料说明 CVM 绑定公网 IP 和 CLB 暴露服务分别适合什么场景。
- 基于腾讯云官方文档，给我一份 COS + CDN 的上传分发方案，并标出前提条件和限制。
- 腾讯云 SCF（云函数）现在支持 Go 1.21 吗？请先查官方文档再回答，并给出 CAM 权限配置。
