# 腾讯云官方文档优先策略

本参考文档用于约束 agent 在处理腾讯云相关任务时的检索顺序、证据来源和输出方式。

## 1. 信源优先级

按以下顺序建立事实：

1. 腾讯云产品文档入口：<https://cloud.tencent.com/document/product>
2. API 参考文档：<https://cloud.tencent.com/document/api>
3. SDK 文档与官方示例：<https://cloud.tencent.com/document/sdk>
4. 官方最佳实践、FAQ、限制说明、计费页面、发布记录
5. 腾讯云开发者社区官方账号发布的技术文章（仅辅助参考，不替代产品文档）
6. 第三方博客、社区帖子、视频、旧教程，仅可作为补充，不可替代官方依据

## 3. 必查信息

处理腾讯云请求时，优先核对以下维度：

- 产品名称是否准确，是否已有改名、合并或升级（如 "CMQ" 已升级为 "TDMQ"）
- 地域、可用区、实例规格、套餐、配额是否受限（金融地域需单独申请）
- 当前推荐的 API/SDK 版本（优先 3.0 SDK）、参数名与鉴权方式
- CAM 权限（主账号/子账号/角色）、VPC 网络、安全组、ACL 等前置条件
- 当前计费模式（包年包月/按量计费/竞价实例）、免费额度与资源上限
- 文档是否存在时间、版本、地域或产品线差异

## 4. 输出格式要求

回答腾讯云问题时，应尽量包含：

1. 结论或推荐方案。
2. 依据官方文档整理出的步骤、参数或代码。
3. 关键限制、风险、前置条件（含 CAM 权限和 VPC 约束）。
4. 不确定项或需要用户二次确认的信息。

如果未直接访问到官方文档，不要假装已经验证过；必须明确说明当前回答的置信度有限。

## 5. 常见误区

- 把云产品旧名称当成当前正式名称（如用 "CDB" 指代所有 TencentDB 系列）
- 忽略地域差异，默认所有地域都支持同一能力
- 按旧控制台截图描述路径，未确认入口是否变更
- 直接复用旧版 SDK 示例（1.x/2.x 接口）或历史 API 参数
- 忽略 CAM 权限、安全组与 VPC 边界，导致方案无法落地
- 只讲主资源费用，忽略带宽、存储、请求次数、API 调用或增值组件成本
- 忽略备案要求，导致大陆节点 HTTP/HTTPS 服务无法对外提供

## 6. 建议检索词

检索时可组合以下信息：

- 产品名 + 功能名
- 产品名 + 地域/实例规格/计费
- 产品名 + API/SDK/CLI/Terraform
- 产品名 + 限制/配额/FAQ/错误码
- 产品名 + 最佳实践/发布记录
- 产品名 + CAM 授权/权限策略

## 7. 何时需要再次检索

出现以下情况时，应重新查官方文档：

- 用户明确要求"最新""当前""现在"
- 用户提供的资料时间明显较早
- 答案涉及价格、限制、配额、地域、控制台步骤
- 答案涉及 API、SDK、权限策略、网络配置或安全配置
- 用户反馈实操结果与既有认知不一致
- 产品名称或功能描述不确定（腾讯云产品命名变更频繁）
