# 阿里云官方文档优先策略

本参考文档用于约束 agent 在处理阿里云相关任务时的检索顺序、证据来源和输出方式。

## 1. 信源优先级

按以下顺序建立事实：

1. 阿里云帮助中心与产品文档：<https://help.aliyun.com/>
2. 对应产品的 API 参考文档与 OpenAPI 页面
3. 对应语言的官方 SDK 文档、官方代码示例与 CLI 文档
4. 官方最佳实践、FAQ、限制说明、计费页面、发布记录
5. 第三方博客、社区帖子、视频、旧教程，仅可作为补充，不可替代官方依据

## 2. 必查信息

处理阿里云请求时，优先核对以下维度：

- 产品名称是否准确，是否已有改名、合并或升级
- 地域、可用区、实例规格、套餐、配额是否受限
- 当前推荐的 API/SDK 版本、参数名与鉴权方式
- RAM 权限、VPC 网络、白名单、安全组等前置条件
- 当前计费模式、价格构成、资源上限与可能的隐性成本
- 文档是否存在时间、版本、地域或产品线差异

## 3. 输出格式要求

回答阿里云问题时，应尽量包含：

1. 结论或推荐方案。
2. 依据官方文档整理出的步骤、参数或代码。
3. 关键限制、风险、前置条件。
4. 不确定项或需要用户二次确认的信息。

如果未直接访问到官方文档，不要假装已经验证过；必须明确说明当前回答的置信度有限。

## 4. 常见误区

- 把云产品旧名称当成当前正式名称
- 忽略地域差异，默认所有地域都支持同一能力
- 按旧控制台截图描述路径，未确认入口是否变更
- 直接复用旧版 SDK 示例或历史 API 参数
- 忽略 RAM 权限、网络边界与安全组，导致方案无法落地
- 只讲主资源费用，忽略带宽、存储、请求次数或增值组件成本

## 5. 建议检索词

检索时可组合以下信息：

- 产品名 + 功能名
- 产品名 + 地域/实例规格/计费
- 产品名 + API/SDK/CLI/Terraform
- 产品名 + 限制/配额/FAQ/错误码
- 产品名 + 最佳实践/发布记录

## 6. 何时需要再次检索

出现以下情况时，应重新查官方文档：

- 用户明确要求“最新”“当前”“现在”
- 用户提供的资料时间明显较早
- 答案涉及价格、限制、配额、地域、控制台步骤
- 答案涉及 API、SDK、权限策略、网络配置或安全配置
- 用户反馈实操结果与既有认知不一致
