# Dependency Range Cheatsheet

跨生态范围表达速查（离线）。

## npm / node-semver

- `^1.2.3` => `>=1.2.3 <2.0.0`
- `^0.2.3` => `>=0.2.3 <0.3.0`
- `~1.2.3` => `>=1.2.3 <1.3.0`
- `1.2.x` => `>=1.2.0 <1.3.0`
- `1.x` => `>=1.0.0 <2.0.0`

注意：预发布版本默认不被普通范围吸收，除非范围显式包含对应 pre-release 语义或设置 include-prerelease。

## Cargo (Rust)

- `1.2.3`（默认 caret）=> `>=1.2.3 <2.0.0`
- `0.2.3` => `>=0.2.3 <0.3.0`
- `~1.2.3` => `>=1.2.3 <1.3.0`
- `1.*` => `>=1.0.0 <2.0.0`

关键差异：Cargo 在 `0.y.z` 阶段的兼容策略与“纯 SemVer 口语理解”常有偏差，需按 Cargo 文档解释。

## Python (PEP 440)

- `~=2.2` 近似 `>=2.2, ==2.*`
- `~=1.4.5` 近似 `>=1.4.5, ==1.4.*`
- `==1.2.*` 前缀匹配
- `!=1.2.*` 前缀排除

注意：Python 使用 PEP 440 体系，不等同 npm/Cargo 的 `^`、`~` 规则。

## Range Design Recommendations

- Library：避免锁死到单一 patch（除非安全/兼容强约束）
- App：可更严格以提升可复现性
- 对已知坏版本使用 `!=` 显式排除
- 预发布依赖应显式说明风险

## Common Pitfalls

- 将 `^0.x` 理解为“可随便升级到 1.0 前”
- 将 npm `~` 与 Python `~=` 视为同一语义
- 在没有解释前提下混用多生态范围符号
