# 符号与字符规范

## 原则

- 优先使用纯 ASCII 符号（`[ ]`, `[OK]`, `->`,  `-`, `+`, `|` 等）以保证跨平台一致性。
- 禁止使用表情符 Emoji（✅, ❌, 📋 等）。
- 禁止使用装饰性 Unicode（如 Unicode 边框 ┌─┐ 或特殊符号 ★ ☆ ♡）。
- 在中文或多语言内容中，可酌情使用汉字或拉丁字符来构建表达，但维持清晰和一致。

## 2. 符号映射表

### 2.1 任务状态符号

| 符号    | 含义         | 使用场景           |
| ------- | ------------ | ------------------ |
| `[ ]`   | 未选中       | 任务列表、复选框   |
| `[x]`   | 选中         | 任务完成标记       |
| `[~]`   | 半选/中间态  | 部分完成           |
| `[*]`   | 当前项/活动项 | 焦点项             |
| `[+]`   | 展开状态     | 树形结构展开节点   |
| `[-]`   | 折叠状态     | 树形结构折叠节点   |

### 2.2 状态标记

| 符号     | 含义   | 使用场景       |
| -------- | ------ | -------------- |
| `[OK]`   | 成功   | 任务完成       |
| `[FAIL]` | 失败   | 任务失败       |
| `[WARN]` | 警告   | 需要注意       |
| `[INFO]` | 信息   | 提示信息       |
| `[>>]`   | 进行中 | 正在处理       |
| `[?]`    | 待处理 | 需要检查       |
| `[SKIP]` | 跳过   | 已跳过         |

### 2.3 列表符号

| 符号        | 含义      | 使用场景       |
| ----------- | --------- | -------------- |
| `-`         | 无序列表  | Markdown 列表  |
| `1.` `2.` `3.` | 有序列表  | 有序列表项     |
| `+`         | 子项/缩进 | 层级结构       |

### 2.4 箭头符号

| 符号    | 含义   | 使用场景     |
| ------- | ------ | ------------ |
| `->`    | 右箭头 | 流程或导航   |
| `<-`    | 左箭头 | 反向流程     |
| `/\`    | 上箭头 | 向上方向     |
| `\/`    | 下箭头 | 向下方向     |
| `<->`   | 双向箭头 | 双向流程   |

### 2.5 其他符号

| 符号         | 含义       | 使用场景           |
| ------------ | ---------- | ------------------ |
| `-----`      | 分隔线     | 区块分隔           |
| `=====`      | 标题下划线 | Markdown 标题样式  |
| `+ - \|`     | 表格边框   | 经典 ASCII 表格    |
| `[====>    ]` | 进度条     | 用 `=` 和空格填充  |
| `@`          | 块填充/像素 | 深色填充           |
| `. : , o`    | 点阵/阴影  | 细节效果           |
| `!!!`        | 强调       | 视觉强调           |
| `\|`         | 竖线       | 垂直分隔           |
| `/ \`        | 斜线       | 对角线             |

## 4. 使用示例

### 4.1 状态标记示例

```
[ ] Write tests
[x] Implement feature A
[-] Review PR #42
[*] Current task
[>>] In progress task
[SKIP] Skip this test
```

### 4.2 树状结构示例

```
root
+-- src
|   +-- index.js
|   +-- utils.js
|   \-- config.json
+-- test
|   +-- main.test.js
\-- README.md
```

### 4.3 流程图示例

```
Request -> Process -> Response
   ^                    |
   |____________________|

[Input] --> [Filter] --> [Output]
              |
              v
            [Log]

Start
  |--> Path A (success)
  \--> Path B (error)
```

### 4.4 表格示例

```
+----+----------+-------+
| ID | Name     | Size  |
+----+----------+-------+
| 1  | alpha    | 12KB  |
| 2  | beta     | 8KB   |
| 3  | gamma    | 15KB  |
+----+----------+-------+
```

### 4.5 进度条示例

```
Progress: [###########       ] 55%
Loading:  [######            ] 30%
Complete: [###################] 100%
```

### 4.6 日志输出示例

```
[INFO] Application started
[OK] Configuration loaded
[WARN] Low disk space
[FAIL] Connection timeout
[>>] Processing request...
[x] Task completed
```

### 4.7 测试报告示例
