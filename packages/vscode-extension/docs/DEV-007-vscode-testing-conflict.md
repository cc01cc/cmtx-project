# DEV-007: 冲突处理配置测试清单

## 测试环境准备

1. 确保 VS Code 扩展已编译并安装
2. 配置云存储（阿里云 OSS 或腾讯云 COS）
3. 准备测试 Markdown 文件
4. **重要**：冲突处理已改为配置驱动，请在 `cmtx.json` 中设置 `upload.conflictStrategy`

## 配置说明

冲突处理策略现在通过配置文件控制：

```json
{
    "upload": {
        "conflictStrategy": "skip" // 或 "overwrite"
    }
}
```

- `"skip"`：跳过已存在的文件（保留远程版本）
- `"overwrite"`：覆盖已存在的文件（上传本地版本）

## 测试场景

### 场景 1：单文件冲突（配置驱动）

**目的**：验证单个文件冲突时的配置驱动处理

**步骤**：

1. 上传一个图片到云存储（例如 `image-1.png`）
2. 在 Markdown 中引用该图片的本地路径：`![alt](image-1.png)`
3. 设置 `conflictStrategy: "skip"` 或 `"overwrite"`
4. 执行上传命令

**预期结果**：

- [ ] 根据配置自动跳过或覆盖文件
- [ ] 不显示冲突对话框
- [ ] 在 Output Channel 和 Debug Console 中显示处理日志
- [ ] 显示统计信息：处理 X 个文件，跳过/覆盖 X 个文件

### 场景 2：多文件冲突（配置驱动）

**目的**：验证多个文件冲突时的配置驱动处理

**步骤**：

1. 上传多个图片到云存储（例如 `image-1.png`, `image-2.png`, `image-3.png`）
2. 在 Markdown 中引用这些图片的本地路径
3. 设置 `conflictStrategy: "skip"` 或 `"overwrite"`
4. 执行上传命令

**预期结果**：

- [ ] 根据配置自动跳过或覆盖所有文件
- [ ] 不显示冲突对话框
- [ ] 在 Output Channel 和 Debug Console 中显示详细日志
- [ ] 显示统计信息：处理 X 个文件，跳过/覆盖 X 个文件

### 场景 3：全选/取消功能

**目的**：验证全选按钮的状态流转

**步骤**：

1. 触发多文件冲突对话框
2. 观察全选按钮初始状态（应为 `[x] 取消`，因为默认全选）
3. 点击"取消"按钮
4. 观察状态变化
5. 点击"全选"按钮

**预期结果**：

- [ ] 初始状态：`[x] 取消`（全选中）
- [ ] 点击"取消"后：所有复选框取消选中，按钮变为 `[ ] 全选`
- [ ] 点击"全选"后：所有复选框选中，按钮变为 `[x] 取消`
- [ ] 手动取消部分选择后：按钮变为 `[~] 全选`

### 场景 4：批量操作

**目的**：验证批量操作功能

**步骤**：

1. 触发多文件冲突对话框（3 个文件）
2. 选择其中 2 个文件（取消第 3 个）
3. 点击"跳过选中"

**预期结果**：

- [ ] 选中的 2 个文件被跳过
- [ ] 未选中的 1 个文件保持原样
- [ ] Markdown 中对应的 2 个图片未被替换

### 场景 5：无选中操作

**目的**：验证未选择文件时的提示

**步骤**：

1. 触发多文件冲突对话框
2. 点击"取消全选"
3. 点击"跳过"或"替换"按钮

**预期结果**：

- [ ] 显示警告提示："请先选择要处理的文件"
- [ ] 对话框不关闭，等待用户选择

### 场景 6：取消操作

**目的**：验证取消对话框功能

**步骤**：

1. 触发多文件冲突对话框
2. 按 ESC 键或点击对话框外部关闭

**预期结果**：

- [ ] 对话框关闭
- [ ] 返回 `undefined`
- [ ] 不执行任何操作

### 场景 7：回车键确认

**目的**：验证默认操作（回车键）

**步骤**：

1. 触发多文件冲突对话框
2. 按回车键

**预期结果**：

- [ ] 对话框关闭
- [ ] 默认执行"替换"操作
- [ ] 选中的文件被替换

### 场景 8：混合场景

**目的**：验证文件部分存在的情况

**步骤**：

1. 上传部分文件到云存储（例如 `image-1.png` 存在，`image-2.png` 不存在）
2. 在 Markdown 中引用这两个图片
3. 执行上传命令

**预期结果**：

- [ ] 存在的文件显示在冲突对话框中
- [ ] 不存在的文件正常上传
- [ ] 最终结果显示：1 个上传，1 个根据用户选择处理
