# 测试最佳实践 (始终生效)

## 测试框架

- 使用 Vitest 作为测试运行器
- 支持 TypeScript 原生测试
- 实现并行测试执行
- 提供代码覆盖率报告

## 测试组织规范

### 目录结构

```
tests/
├── unit/           # 单元测试
├── integration/    # 集成测试  
├── e2e/           # 端到端测试
└── fixtures/      # 测试数据
```

### 命名约定

- 测试文件：*.test.ts
- 测试数据：*.fixture.ts
- 描述格式：describe('模块名', () => {...})

### 测试内容要求

- 每个公共函数都应有对应测试
- 覆盖正常路径和异常路径
- 测试边界条件和特殊情况
- 验证错误处理机制

## Mock 和 Stub 策略

- 使用 vi.mock() 进行模块模拟
- 实现假数据生成器
- 提供测试工具函数
- 确保测试环境隔离

## 断言最佳实践

- 使用 expect 进行断言
- 提供清晰的断言消息
- 验证返回值和副作用
- 检查异步操作结果
