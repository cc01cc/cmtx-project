# 代码审查清单 (手动引入)

## 使用场景

代码审查、Pull Request 检查、质量保证时手动引入

## 功能完整性检查

### [ ] 核心功能实现

- [ ] 所有声明的公共 API 都已实现
- [ ] 边界条件和异常情况已处理
- [ ] 返回值类型与文档描述一致
- [ ] 异步操作正确处理 Promise

### [ ] 错误处理

- [ ] 关键路径都有适当的错误处理
- [ ] 错误信息清晰且有助于调试
- [ ] 没有未处理的 Promise rejection
- [ ] 资源泄露风险已评估和处理

## 代码质量检查

### [ ] 代码风格

- [ ] 符合 TypeScript 严格模式要求
- [ ] 命名规范一致且具有描述性
- [ ] 适当的注释和文档字符串
- [ ] 复杂度合理的函数和类

### [ ] 性能考量

- [ ] 没有明显的性能瓶颈
- [ ] 适当的缓存和优化策略
- [ ] 大数据处理使用流式或分批处理
- [ ] 内存使用在合理范围内

## 测试覆盖检查

### [ ] 测试完整性

- [ ] 新增功能都有对应的测试用例
- [ ] 测试覆盖了正常和异常路径
- [ ] 边界条件和特殊情况已测试
- [ ] 集成测试验证整体功能

### [ ] 测试质量

- [ ] 测试用例命名清晰描述测试目的
- [ ] 测试数据具有代表性和多样性
- [ ] Mock 和 Stub 使用合理
- [ ] 测试执行速度快且稳定

## 安全性检查

### [ ] 安全实践

- [ ] 输入验证和净化已实施
- [ ] 敏感信息处理符合安全要求
- [ ] 没有硬编码的凭证或密钥
- [ ] 文件路径和系统调用已验证

### [ ] 依赖安全

- [ ] 第三方依赖版本经过安全审查
- [ ] 没有已知的安全漏洞依赖
- [ ] 依赖的许可证兼容性已确认

## 文档和可维护性

### [ ] 文档完善

- [ ] API 文档完整且准确
- [ ] README 更新反映最新变化
- [ ] 配置选项和环境变量有说明
- [ ] 示例代码可正常运行

### [ ] 可维护性

- [ ] 代码结构清晰，职责分离良好
- [ ] 复杂逻辑有足够的注释解释
- [ ] 魔法数字和字符串已提取为常量
- [ ] 未来扩展性已考虑
